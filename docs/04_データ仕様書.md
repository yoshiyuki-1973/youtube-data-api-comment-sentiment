# データ仕様書
YouTube分析バッチシステム

## 1. データ構造

本システムではデータの永続化（ファイル保存・DB保存）は行わず、
YouTube APIから取得したデータをメモリ上で処理し、結果を画面に表示する。

### 1.1 動画情報（メモリ上の辞書）
```json
{
  "video_id": "string",
  "title": "string",
  "channel_id": "string",
  "channel_title": "string",
  "published_at": "string (ISO 8601)",
  "view_count": "integer",
  "like_count": "integer",
  "comment_count": "integer",
  "fetched_at": "string (ISO 8601)"
}
```

### 1.2 コメント情報（メモリ上のリスト）
```json
{
  "comment_id": "string",
  "author": "string",
  "text": "string",
  "like_count": "integer",
  "published_at": "string (ISO 8601)",
  "sentiment": {
    "positive": "float (0.0-1.0)",
    "negative": "float (0.0-1.0)",
    "neutral": "float (0.0-1.0)",
    "language": "string (ja/other)"
  }
}
```

## 2. データフロー

本システムのデータフローは以下の通り：

```
YouTube Data API → データ取得 → 感情分析 → 集計 → 画面表示（保存なし）
```

- すべてのデータはメモリ上で処理される
- 実行のたびにYouTube APIからデータを取得する
- 分析結果はStreamlit UIまたはCLI出力で表示される
