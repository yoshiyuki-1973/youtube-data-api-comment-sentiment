# 詳細設計書

YouTube動画コメント感情分析システム

## 1. 設計方針

- I/Oとロジックを分離し、外部依存をモックでテスト可能にする
- main処理は薄く保ち、各モジュールに責務を委譲する
- 型ヒントを全関数に付与する
- Google Style docstringで関数を文書化する

## 2. モジュール構成

```
app/
├── __init__.py          # パッケージ初期化
├── main.py              # CLIエントリーポイント
├── streamlit_app.py     # Streamlit WebUIエントリーポイント
├── fetch/               # データ取得層
│   ├── __init__.py
│   └── youtube.py       # YouTube Data API クライアント
├── sentiment/           # 感情分析層
│   ├── __init__.py
│   └── analyzer.py      # 3-model アンサンブル感情分析エンジン
└── aggregate/           # 集計層
    ├── __init__.py
    └── summarizer.py    # コメント集計・統計計算
```

## 3. 関数仕様

### 3.1 fetch/youtube.py

#### カスタム例外

```python
class YouTubeAPIError(Exception):
    """YouTube API エラーの基底クラス"""

class QuotaExceededError(YouTubeAPIError):
    """APIクォータ超過（403 quotaExceeded）"""

class AuthenticationError(YouTubeAPIError):
    """APIキー無効・アクセス拒否（401/403）"""

class VideoNotFoundError(YouTubeAPIError):
    """動画が見つからない（404）"""

class CommentsDisabledError(YouTubeAPIError):
    """コメントが無効化されている（403 commentsDisabled）"""
```

#### 主要関数

```python
def _get_client():
    """
    YouTube APIクライアントを取得または生成する（遅延初期化）

    Returns:
        googleapiclient.discovery.Resource: YouTube APIクライアント

    """

def _handle_http_error(e: HttpError, context: str) -> None:
    """
    HttpErrorを解析し適切なカスタム例外を送出する

    Args:
        e: HttpErrorインスタンス
        context: エラー発生箇所の説明文

    Raises:
        QuotaExceededError / AuthenticationError /
        VideoNotFoundError / CommentsDisabledError / YouTubeAPIError
    """

def fetch_video(video_id: str) -> dict | None:
    """
    動画メタ情報を取得する

    Args:
        video_id: YouTube動画ID

    Returns:
        動画情報の辞書（以下のキー）
        {
            "video_id": str,
            "title": str,
            "channel_id": str,
            "channel_title": str,
            "published_at": str (ISO 8601),
            "view_count": int,
            "like_count": int,
            "comment_count": int,
            "fetched_at": str (ISO 8601)
        }
        取得失敗時はNone
    """

def fetch_comments(video_id: str, comment_limit: int = 10) -> list[dict]:
    """
    コメントを取得する（いいね数降順ソート）

    Args:
        video_id: YouTube動画ID
        comment_limit: 取得件数（デフォルト10）

    Returns:
        コメント情報のリスト。各要素:
        {
            "comment_id": str,
            "author": str,
            "text": str,
            "like_count": int,
            "published_at": str (ISO 8601)
        }

    Notes:
        - APIはrelevance順で取得後、like_count降順でソート
        - COMMENT_FETCH_MULTIPLIER倍のコメントを取得して上位を返す
        - ページネーション対応（nextPageToken）
    """
```

_ja_model_1 = None       # 日本語モデル1（christian-phu: 3クラス）

#### ルールベース定数

| 定数名 | パターン数 | 用途 |
|--------|-----------|------|
| `POSITIVE_WORDS` | 50+ | ポジティブキーワード（日英+絵文字） |
| `NEGATIVE_WORDS` | 50+ | ネガティブキーワード（日英+絵文字） |
| `STRONG_NEGATIVE_PATTERNS` | 100+ | 強いネガティブ表現（YouTube特有含む） |
| `STRONG_POSITIVE_PATTERNS` | 100+ | 強いポジティブ表現 |
| `SARCASM_PATTERNS` | 20+ | 皮肉・棒読み検出（正規表現） |
| `RHETORICAL_PATTERNS` | 20+ | 反語表現検出（正規表現） |
| `NEGATION_PATTERNS` | 15+ | 否定表現（感情反転用） |

#### 主要関数

```python
def load_models() -> None:
    """
    感情分析モデルをロードする（起動時に1回のみ、スレッドセーフ）

    Notes:
        - Double-checked locking パターンを使用
        - 3つのモデルを順次ロード
        - 個別のモデルロード失敗はログ出力し、残りのモデルは継続
        - モデルは評価モード（eval()）に設定
    """

def _detect_language(text: str) -> str:
    """
    テキストの言語を検出する

    Args:
        text: 入力テキスト

    Returns:
        "ja"（日本語）または "other"（その他）

    Notes:
        - まず文字ベース判定（ひらがな/カタカナ/漢字）
        - 日本語文字がなければ langdetect にフォールバック
    """

def _preprocess_text(text: str) -> str:
    """
    テキストの前処理を行う

    処理内容:
        1. URL除去（https?://...）
        2. HTMLタグ除去（<...>）
        3. 空白正規化（連続空白を1つに）

    Args:
        text: 入力テキスト

    Returns:
        前処理済みテキスト
    """

def _rule_based_classify(text: str) -> str:
    """
    ルールベースの感情分類（バイナリ: pos/neg）

    Args:
        text: 入力テキスト

    Returns:
        "pos" または "neg"

    Notes:
        - POSITIVE_WORDS / NEGATIVE_WORDS のマッチ数で判定
        - 同数の場合は "pos" を返す
    """

def _adjust_sentiment_with_rules(text: str, scores: dict) -> dict:
    """
    ルールベースによるスコア補正

    Args:
        text: 元テキスト
        scores: モデル予測スコア {"positive", "negative", "neutral"}

    Returns:
        補正後スコア（正規化済み、合計1.0）

    補正ルール:
        1. 強いネガティブ表現 → negative +0.15~0.25
        2. 皮肉表現 → negative +0.2
        3. 反語表現 → negative +0.2
        4. ポジティブ+否定 → negative +0.2
        5. 強いポジティブ表現 → positive +0.15~0.25
        ※ 補正値は最大 +/-0.3 に制限
    """

def _single_model_inference(text: str, model, tokenizer, id2label) -> dict:
    """
    単一モデルでの推論

    Args:
        text: 前処理済みテキスト
        model: PyTorchモデル
        tokenizer: トークナイザ
        id2label: ラベルマッピング辞書

    Returns:
        {"positive": float, "negative": float, "neutral": float}
        推論失敗時はNone

    Notes:
        - 3クラスモデル: id2labelに基づきスコアをマッピング
        - 2クラスモデル: neutralは0.0として返す
    """

def _pytorch_inference(text: str, language: str) -> dict:
    """
    言語に応じたモデルで推論

    Args:
        text: 前処理済みテキスト
        language: "ja" または "other"

    Returns:
        {"positive": float, "negative": float, "neutral": float}

    Notes:
        - 日本語: 2モデルのスコア平均（アンサンブル）
        - その他: 多言語モデル単体
        - 全モデル失敗時はルールベースにフォールバック
    """

def classify_comment(text: str) -> dict:
    """
    コメントの感情を分類する

    Args:
        text: コメント本文

    Returns:
        {
            "positive": float,   # 0.0-1.0
            "negative": float,   # 0.0-1.0
            "neutral": float,    # 0.0-1.0
            "language": str      # "ja" | "other" | "unknown"
        }

    Notes:
        - 空文字列/Noneの場合は均等分布（0.33/0.33/0.34）を返す
        - 前処理 → 言語検出 → モデル推論 → スコア返却
    """

def _classify_comment_rules_only(text: str) -> dict:
    """
    ルールベースのみで感情分類する（フォールバックモード）

    Args:
        text: コメント本文

    Returns:
        {"positive": float, "negative": float, "neutral": float, "language": str}

    Notes:
        - 全モデル失敗時かつFALLBACK_TO_RULES_ONLY=true時に使用
        - ルールベース分類 + ルール補正を適用
    """

def classify_comments(comments: list[dict]) -> list[dict]:
    """
    コメントリストに感情分類を一括付与する

    Args:
        comments: コメント情報のリスト

    Returns:
        各コメントに'sentiment'フィールドが追加されたリスト

    Raises:
        RuntimeError: 全モデル失敗かつFALLBACK_TO_RULES_ONLY=false時

    Notes:
        - 初回呼び出し時にモデルをロード
        - 順次処理（並列処理なし）
        - FALLBACK_TO_RULES_ONLY=true で全モデル失敗時もルールベースで継続
    """
```

### 3.3 aggregate/summarizer.py

```python
def aggregate_video(video: dict, comments: list[dict]) -> dict:
    """
    動画単位で感情分析結果を集計する

    Args:
        video: 動画情報の辞書
        comments: 感情分類済みコメントのリスト

    Returns:
        {
            "video_id": str,
            "total_comments": int,
            "positive_count": int,      # 支配的感情がpositiveの件数
            "negative_count": int,      # 支配的感情がnegativeの件数
            "other_count": int,         # neutral/同スコアの件数
            "positive_ratio": float,    # positive件数比率（4桁精度）
            "negative_ratio": float,    # negative件数比率（4桁精度）
            "positive_score": float,    # 平均positiveスコア（4桁精度）
            "negative_score": float,    # 平均negativeスコア（4桁精度）
            "neutral_score": float,     # 平均neutralスコア（4桁精度）
            "analyzed_at": str          # ISO 8601形式
        }

    Notes:
        - 同スコアの場合は "other"（あいまい）に分類
        - コメント0件の場合は全フィールド0を返す
    """
```

## 4. 命名規約

### 4.1 識別子

| 種類 | 規則 | 例 |
|------|------|-----|
| 変数 | snake_case | `video_id`, `comment_limit` |
| 関数 | snake_case | `fetch_comments()`, `classify_comment()` |
| クラス | PascalCase | `YouTubeAPIError`, `QuotaExceededError` |
| 定数 | UPPER_SNAKE_CASE | `MAX_LENGTH`, `API_MAX_RESULTS` |
| モジュール | snake_case | `youtube.py`, `analyzer.py` |
| プライベート | _prefix | `_get_client()`, `_model_lock` |

### 4.2 型ヒント

```python
# 必須: 関数の引数と戻り値に型ヒントを付ける
def fetch_comments(video_id: str, comment_limit: int = 10) -> list[dict]:
    ...
```

### 4.3 感情ラベル

| 内部ラベル | 表示名 | 意味 |
|-----------|--------|------|
| `pos` | ポジティブ | 肯定的なコメント |
| `neg` | ネガティブ | 否定的なコメント |
| `other` | ニュートラル/その他 | 中立・判定困難 |

### 4.4 データキー命名

| 規則 | 例 |
|------|-----|
| snake_case | `video_id`, `comment_count` |
| タイムスタンプは `*_at` | `fetched_at`, `analyzed_at`, `published_at` |
| スコアは `*_score` | `positive_score`, `negative_score` |
| 比率は `*_ratio` | `positive_ratio`, `negative_ratio` |
| 件数は `*_count` | `positive_count`, `total_comments` |

### 4.5 ファイル命名

| 種類 | 規則 | 例 |
|------|------|-----|
| ソースモジュール | snake_case.py | `youtube.py`, `analyzer.py` |
| テストファイル | `test_{対象モジュール}.py` | `test_analyzer.py` |
| 設定ファイル | 小文字 | `pytest.ini`, `docker-compose.yml` |
| 設計書 | `{番号}_{名称}.md` | `02_要件定義書.md` |
| ログファイル | `{種類}.log` | `app.log`, `error.log` |

### 4.6 環境変数

| 規則 | 例 |
|------|-----|
| UPPER_SNAKE_CASE | `YOUTUBE_API_KEY`, `COMMENT_LIMIT` |
| プレフィックスでグループ化 | `YOUTUBE_*`, `COMMENT_*` |

### 4.7 ログメッセージ

```
[レベル] モジュール名: 動作 + 対象

例:
INFO fetch: Fetching video: dQw4w9WgXcQ
INFO sentiment: Analyzed 10 comments for video: dQw4w9WgXcQ
ERROR fetch: API quota exceeded (403)
WARNING aggregate: Duplicate video_id, updating: dQw4w9WgXcQ
```
