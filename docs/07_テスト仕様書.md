# テスト仕様書

YouTube動画コメント感情分析システム

## 1. テスト方針

- pytestフレームワークを使用
- 外部I/O（YouTube API, Hugging Faceモデル）はモック化
- テストデータはfixturesで管理
- 単体テスト + 統合テストの二段構成

### 1.1 カバレッジ目標

| 対象 | 目標 | 備考 |
|------|------|------|
| 全体 | 80%以上 | Line Coverage基準 |
| ビジネスロジック | 90%以上 | sentiment/, aggregate/ |
| I/O層 | 70%以上 | fetch/（モック前提） |

**除外対象**: 設定ファイル、`if __name__ == "__main__"` ブロック

## 2. テスト構成

### 2.1 テストファイル一覧

| ファイル | 対象モジュール | テスト数 | 内容 |
|---------|--------------|---------|------|
| `test_analyzer.py` | sentiment/analyzer.py | 20+ | 感情分析エンジン |
| `test_fetch.py` | fetch/youtube.py | 12+ | YouTube API取得 |
| `test_summarizer.py` | aggregate/summarizer.py | 10+ | 集計処理 |
| `test_integration.py` | 複合 | 8+ | パイプライン統合テスト |
| `conftest.py` | - | - | 共通フィクスチャ |

### 2.2 テスト実行コマンド

```bash
# 全テスト実行
docker compose exec app pytest

# 詳細出力付き
docker compose exec app pytest -v

# 単一ファイル実行
docker compose exec app pytest tests/test_analyzer.py -v

# 特定テスト関数実行
docker compose exec app pytest tests/test_analyzer.py::TestClassifyComment::test_classify_positive -v

# カバレッジ付き実行
docker compose exec app pytest --cov=app --cov-report=html

# 失敗したテストのみ再実行
docker compose exec app pytest --lf

# 最初の失敗で停止
docker compose exec app pytest -x
```

## 5. モック設計

### 5.1 モック対象一覧

| 対象 | モック方法 | 用途 |
|------|-----------|------|
| YouTube Data API | `@patch('fetch.youtube._get_client')` | API呼び出しの代替 |
| Transformersモデル | `@patch('sentiment.analyzer._ja_model_1')` 等 | モデル推論の代替 |
| load_models関数 | `@patch('sentiment.analyzer.load_models')` | モデルロード回避 |
| 環境変数 | `monkeypatch.setenv()` | 環境変数切り替え |

### 5.3 YouTube Data API モック例

```python
from unittest.mock import patch, MagicMock

@patch('fetch.youtube._get_client')
def test_fetch_video_success(mock_get_client):
    mock_api = MagicMock()
    mock_get_client.return_value = mock_api

    mock_response = {
        'items': [{
            'id': 'dQw4w9WgXcQ',
            'snippet': {
                'title': 'Test Video',
                'channelId': 'UC123',
                'channelTitle': 'Test Channel',
                'publishedAt': '2025-01-01T00:00:00Z'
            },
            'statistics': {
                'viewCount': '1000',
                'likeCount': '100',
                'commentCount': '10'
            }
        }]
    }
    mock_api.videos.return_value.list.return_value.execute.return_value = mock_response

    result = fetch_video('dQw4w9WgXcQ')
    assert result['video_id'] == 'dQw4w9WgXcQ'
```

### 5.4 Transformers モック例

```python
from unittest.mock import patch, MagicMock
import torch

@patch('sentiment.analyzer._ja_model_1')
@patch('sentiment.analyzer._ja_tokenizer_1')
def test_classify_with_mock(mock_tokenizer, mock_model):
    mock_tokenizer.return_value = {
        'input_ids': torch.tensor([[1, 2, 3]]),
        'attention_mask': torch.tensor([[1, 1, 1]])
    }

    mock_output = MagicMock()
    mock_output.logits = torch.tensor([[0.1, 0.1, 0.8]])
    mock_model.return_value = mock_output

    result = classify_comment('素晴らしい動画')
    assert result['positive'] > 0.5
```
