# 基本設計書

YouTube動画コメント感情分析システム

## 1. システム全体構成

```
ユーザー
  │
  ├── Streamlit WebUI (http://localhost:8501)
  │     └── streamlit_app.py
  │
  └── CLI (docker compose exec)
        └── main.py
              │
              ▼
        ┌─────────────────┐
        │  fetch/          │  YouTube Data API v3
        │  youtube.py      │◄────── 動画メタデータ・コメント取得
        └────────┬────────┘
                 │
                 ▼
        ┌─────────────────┐
        │  sentiment/      │  3-model BERT Ensemble
        │  analyzer.py     │  + ルールベース(200+パターン)
        └────────┬────────┘
                 │
                 ▼
        ┌─────────────────┐
        │  aggregate/      │  感情スコア集計
        │  summarizer.py   │
        └────────┬────────┘
                 │
                 ▼
          画面表示（保存なし）
```

## 2. Docker構成

### 2.1 コンテナ一覧

| サービス名 | コンテナ名 | イメージ | 役割 |
|-----------|-----------|---------|------|
| streamlit | youtube-analytics-streamlit | python:3.11-slim | Streamlit WebUI |
| app | youtube-analytics-app | python:3.11-slim | CLIバッチ処理 |

### 2.2 docker-compose.yml 設計

```yaml
services:
  streamlit:
    build: .
    container_name: youtube-analytics-streamlit
    command: python -m debugpy --listen 0.0.0.0:5678 -m streamlit run streamlit_app.py --server.port=8501 --server.address=0.0.0.0
    ports:
      - "8501:8501"   # Streamlit UI
      - "5678:5678"   # debugpy リモートデバッグ
    volumes:
      - ./app:/app
      - ./logs:/app/logs
      - ./models:/app/models
    environment:
      - YOUTUBE_API_KEY=${YOUTUBE_API_KEY}
      - COMMENT_LIMIT=${COMMENT_LIMIT}

  app:
    build: .
    container_name: youtube-analytics-app
    command: sleep infinity
    tty: true
    volumes:
      - ./app:/app
      - ./logs:/app/logs
      - ./models:/app/models
    environment:
      - YOUTUBE_API_KEY=${YOUTUBE_API_KEY}
      - COMMENT_LIMIT=${COMMENT_LIMIT}
```

## 5. 処理フロー

### 5.1 Streamlit UIフロー

```
1. ユーザーが動画ID/URLを入力
2. extract_video_id() でURLから動画IDを抽出
3. 「分析開始」ボタン押下
4. fetch_video() で動画メタデータ取得
5. fetch_comments() でコメント取得（いいね数順ソート）
6. classify_comments() で感情分析（3-model ensemble）
7. aggregate_video() で集計
8. Streamlit UIに結果表示
   - 動画情報（メトリクスカード）
   - 感情サマリー（棒グラフ + スコア）
   - コメント一覧（感情フィルタ付き）
```

### 5.2 CLIバッチフロー

```
1. コマンドライン引数を解析（argparse）
   --video-id / --video-ids / --comment-limit
2. 動画IDリストをループ処理
3. 各動画に対して:
   a. fetch_video() で動画メタデータ取得
   b. fetch_comments() でコメント取得
   c. classify_comments() で感情分析
   d. aggregate_video() で集計
   e. ログに結果を出力
4. バッチ完了ログ（成功/失敗件数）
```

### 5.3 感情分析フロー

```
入力テキスト
  ↓
前処理（URL除去、HTMLタグ除去、空白正規化）
  ↓
言語検出
  ├── 日本語（ひらがな/カタカナ/漢字検出）
  │     ↓
  │   日本語モデル1（christian-phu: 3クラス）
  │   日本語モデル2（kit-nlp: 2クラス+皮肉検出）
  │     ↓
  │   アンサンブル（2モデル平均）
  │
  └── その他言語（langdetectフォールバック）
        ↓
      多言語モデル（XLM-RoBERTa: 3クラス）
  ↓
ルールベース補正（200+パターン）
  ↓
正規化（合計1.0）
  ↓
出力: {positive, negative, neutral, language}
```

## 6. エラーハンドリング設計

### 6.1 カスタム例外階層

```
YouTubeAPIError（基底クラス）
  ├── QuotaExceededError     # APIクォータ超過（403）
  ├── AuthenticationError    # APIキー無効（401/403）
  ├── VideoNotFoundError     # 動画未検出（404）
  └── CommentsDisabledError  # コメント無効化（403）
```

### 6.2 エラー対応方針

| エラー種別 | HTTPステータス | 対応 |
|-----------|--------------|------|
| クォータ超過 | 403 (quotaExceeded) | 処理中断、翌日リトライ |
| 認証エラー | 401 / 403 | APIキー確認を促す |
| 動画未検出 | 404 | スキップして次の動画へ |
| コメント無効化 | 403 (commentsDisabled) | 警告表示、スキップ |
| モデルロード失敗 | - | ルールベースにフォールバック |

## 7. 環境変数設計

| 変数名 | 必須 | デフォルト | 説明 |
|--------|------|-----------|------|
| `YOUTUBE_API_KEY` | 必須 | - | YouTube Data API v3キー |
| `COMMENT_LIMIT` | 任意 | 10 | コメント取得件数 |
| `API_MAX_RESULTS` | 任意 | 100 | 1リクエストあたり最大取得数 |
| `COMMENT_FETCH_MULTIPLIER` | 任意 | 2 | ソート用取得倍率 |
| `LOG_DIR` | 任意 | /app/logs | ログ出力先 |
| `USE_FINETUNED_MODEL` | 任意 | false | Fine-tunedモデル使用有無 |
| `FINETUNED_MODEL_PATH` | 任意 | ./models/sentiment-finetuned | Fine-tunedモデルパス |
| `JA_MODEL_1` | 任意 | christian-phu/bert-finetuned-japanese-sentiment | 日本語モデル1 |
| `JA_MODEL_2` | 任意 | kit-nlp/bert-base-japanese-sentiment-irony | 日本語モデル2 |
| `MULTILINGUAL_MODEL` | 任意 | cardiffnlp/twitter-xlm-roberta-base-sentiment | 多言語モデル |
| `MAX_TOKEN_LENGTH` | 任意 | 128 | 最大トークン長（1-512） |
| `FALLBACK_TO_RULES_ONLY` | 任意 | false | 全モデル失敗時ルールベースフォールバック |
