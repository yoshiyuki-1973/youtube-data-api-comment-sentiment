# 操作マニュアル

YouTube動画コメント感情分析システム ユーザー操作ガイド

## 1. 概要

本システムは、YouTube動画のコメントを取得し、AI（3モデルBERTアンサンブル）による感情分析を行い、結果をわかりやすく表示するツールである。以下の2つの操作方法を提供する。

| 操作方法 | 用途 | アクセス方法 |
|---------|------|------------|
| Streamlit WebUI | 対話的な分析・可視化 | ブラウザで http://localhost:8501 |
| CLI（コマンドライン） | バッチ処理・スクリプト連携 | docker compose exec コマンド |

---

## 2. 事前準備

### 2.1 必要なもの

1. **Docker Desktop** がインストール・起動されていること
2. **YouTube Data API v3 の APIキー** を取得済みであること
3. **.envファイル** に APIキーが設定されていること

### 2.2 システムの起動

```powershell
# コンテナを起動
docker compose up -d

# 起動状態を確認
docker compose ps
```

正常に起動すると、以下の2つのコンテナが表示される:

```
NAME                           STATUS
youtube-analytics-streamlit    Up
youtube-analytics-app          Up
```

### 2.3 システムの停止

```powershell
docker compose down
```

## 3. Streamlit WebUI の操作

### 3.1 WebUIへのアクセス

ブラウザで以下のURLにアクセスする:

```
http://localhost:8501
```

### 3.2 動画IDの入力

画面上部の入力欄に、以下のいずれかの形式でYouTube動画を指定する:

| 入力形式 | 例 |
|---------|-----|
| 動画ID（11文字） | `dQw4w9WgXcQ` |
| 通常URL | `https://www.youtube.com/watch?v=dQw4w9WgXcQ` |
| 短縮URL | `https://youtu.be/dQw4w9WgXcQ` |
| Shorts URL | `https://www.youtube.com/shorts/dQw4w9WgXcQ` |
| 埋め込みURL | `https://www.youtube.com/embed/dQw4w9WgXcQ` |

入力すると、動画IDが自動で抽出され「動画ID: `dQw4w9WgXcQ`」と表示される。

### 3.3 分析の実行

1. 動画ID/URLを入力する
2. 「**分析開始**」ボタンをクリックする
3. 分析中はスピナーが表示される（10コメントで約0.5-1秒、100コメントで約5-10秒）
4. 「分析完了!」が表示されたら結果を確認する

### 3.4 結果の見方

#### 動画情報セクション

動画の基本情報がメトリクスカードで表示される:

| 表示項目 | 説明 |
|---------|------|
| 再生回数 | 動画の総再生回数 |
| 高評価数 | 動画の高評価数 |
| コメント数 | 動画の総コメント数 |
| タイトル | 動画タイトル |
| チャンネル | チャンネル名 |
| 公開日 | 動画の公開日時 |

#### 感情分析結果セクション

コメント全体の感情傾向が表示される:

| 表示項目 | 説明 | 値の範囲 |
|---------|------|---------|
| 分析コメント数 | 分析対象のコメント件数 | 整数 |
| ポジティブ | 平均ポジティブスコア | 0.0000 - 1.0000 |
| ネガティブ | 平均ネガティブスコア | 0.0000 - 1.0000 |
| ニュートラル | 平均ニュートラルスコア | 0.0000 - 1.0000 |

棒グラフで3つのスコアが視覚的に表示される。

#### コメント一覧セクション

個別のコメントと感情スコアが一覧表示される:

- **感情フィルタ**: ドロップダウンで「すべて」「ポジティブ優勢」「ネガティブ優勢」「ニュートラル優勢」を選択可能
- **各コメント**: 展開可能なカード形式で表示
  - `[P:0.85 N:0.10 Neu:0.05]` はポジティブ0.85、ネガティブ0.10、ニュートラル0.05を示す
  - いいね数、投稿者名、投稿日時が表示される

### 3.5 コメント取得件数

コメント取得件数は環境変数 `COMMENT_LIMIT` で設定される（デフォルト: 10件）。画面上に「コメント取得数: XX件」と表示される。変更するには `.env` ファイルの `COMMENT_LIMIT` を編集し、コンテナを再起動する。

```bash
# .envファイルを編集
COMMENT_LIMIT=50

# コンテナを再起動
docker compose restart streamlit
```

## 4. CLI（コマンドライン）の操作

### 4.1 単一動画の分析

```bash
docker compose exec app python main.py --video-id dQw4w9WgXcQ
```

### 4.2 コメント取得件数の指定

```bash
docker compose exec app python main.py --video-id dQw4w9WgXcQ --comment-limit 50
```

### 4.3 複数動画の一括分析

```bash
docker compose exec app python main.py --video-ids VIDEO_ID1,VIDEO_ID2,VIDEO_ID3
```

### 4.4 CLI引数一覧

| 引数 | 説明 | 必須 | デフォルト |
|------|------|------|-----------|
| `--video-id` | 単一動画ID | いずれか必須 | - |
| `--video-ids` | カンマ区切りの動画IDリスト | いずれか必須 | - |
| `--comment-limit` | コメント取得件数 | 任意 | 10（環境変数で変更可） |

## 5. エラーメッセージと対処法

### 5.1 WebUIのエラーメッセージ

| エラーメッセージ | 原因 | 対処法 |
|----------------|------|--------|
| YouTube APIの利用上限に達しました | APIクォータ超過（1日10,000ユニット） | 翌日（太平洋時間0時）まで待つ |
| この動画はコメントが無効化されています | 動画のコメント機能が無効 | 別の動画を指定する |
| 動画が見つかりません | 動画IDが無効または非公開 | URLを確認する |
| YouTube APIの認証に失敗しました | APIキーが無効 | `.env`のYOUTUBE_API_KEYを確認する |
| 有効なYouTube動画IDまたはURLを入力してください | URL形式が不正 | 正しいURL形式で入力する |

### 5.2 CLIのエラー

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `YOUTUBE_API_KEY environment variable is not set` | 環境変数未設定 | `.env`にAPIキーを設定しコンテナを再起動 |
| `QuotaExceededError` | APIクォータ超過 | 翌日まで待つ |
| `RuntimeError: 全ての感情分析モデルのロードに失敗` | モデルダウンロード失敗 | ネットワーク接続を確認しコンテナを再起動 |

## 6. 感情スコアの読み方

### 6.1 スコアの意味

各コメントに対して、以下の3つのスコアが0.0-1.0の範囲で算出される。3つのスコアの合計は常に1.0になる。

| スコア | 意味 | 高スコアの例 |
|--------|------|------------|
| positive | 肯定的な感情の度合い | 「最高！面白かった」 |
| negative | 否定的な感情の度合い | 「つまらない、時間の無駄」 |
| neutral | 中立・判定困難の度合い | 「動画を見ました」 |

### 6.2 集計結果の読み方

| 指標 | 説明 |
|------|------|
| positive_count | ポジティブスコアが最も高いコメントの件数 |
| negative_count | ネガティブスコアが最も高いコメントの件数 |
| other_count | ニュートラルまたは同スコアのコメントの件数 |
| positive_score | 全コメントのポジティブスコアの平均値 |
| negative_score | 全コメントのネガティブスコアの平均値 |
| neutral_score | 全コメントのニュートラルスコアの平均値 |

### 6.3 分析の精度について

- **日本語コメント**: 2つのBERTモデルのアンサンブルで高精度に分析
- **英語等のコメント**: XLM-RoBERTaモデルで分析
- **皮肉・反語**: ルールベース（200以上のパターン）で補正
- **短いコメント**: 絵文字やスラングも検出可能
- **注意点**: 128トークン（約85-100文字）を超える部分は切り詰められる
