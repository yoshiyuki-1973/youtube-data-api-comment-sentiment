# 運用手順書

YouTube動画コメント感情分析システム

## 1. 運用方針

- 手動でバッチ実行する（スケジューラ不使用）
- Docker Composeによるコンテナ管理
- Streamlit WebUI + CLIの2つの実行方式を提供

---

## 2. Docker環境構築

### 2.1 コンテナ構成

本システムは2つのコンテナで構成される。

| コンテナ名 | サービス名 | 役割 | ポート |
|-----------|-----------|------|--------|
| youtube-analytics-streamlit | streamlit | Streamlit WebUI | 8501 |
| youtube-analytics-app | app | CLIバッチ処理 | - |

### 2.2 ネットワーク構成

- すべてのコンテナは `youtube-analytics-network` ブリッジネットワークで接続
- コンテナ間は内部ネットワークで通信

### 2.3 ボリュームマウント

| ホスト側 | コンテナ側 | 用途 |
|---------|-----------|------|
| `./app` | `/app` | ソースコード（ホットリロード） |
| `./logs` | `/app/logs` | ログファイル |
| `./models` | `/app/models` | 感情分析モデル |

## 3. Docker基本操作

### 3.1 初回セットアップ

```powershell
# 環境変数の設定
Copy-Item .env.example .env
# .envを編集してYOUTUBE_API_KEYを設定

# ビルドして起動
docker compose up --build -d

# 起動ログを確認
docker compose logs -f
```

### 3.2 起動・停止

```powershell
# すべてのコンテナを起動
docker compose up -d

# すべてのコンテナを停止
docker compose down

# コンテナとボリュームを削除
docker compose down -v

# 特定のコンテナを再起動
docker compose restart streamlit
```

### 3.3 状態確認

```powershell
# 実行中のコンテナ一覧
docker compose ps

# すべてのコンテナのログを表示
docker compose logs

# 特定のコンテナのログ（リアルタイム）
docker compose logs -f streamlit
docker compose logs -f app
```

### 3.4 イメージ再ビルド

コードを変更した場合、イメージを再ビルドする。

```powershell
# 再ビルド
docker compose build

# キャッシュを使わずに再ビルド
docker compose build --no-cache

# 再ビルドして起動
docker compose up --build -d
```

### 3.5 コンテナ内操作

```powershell
# コンテナ内でシェルを起動
docker compose exec app /bin/bash

# コンテナ内でPythonスクリプトを実行
docker compose exec app python main.py --video-id <VIDEO_ID>

# コンテナ内でpytestを実行
docker compose exec app pytest
```

## 4. バッチ実行

### 4.1 単一動画の分析

```bash
docker compose exec app python main.py --video-id VIDEO_ID
```

### 4.2 コメント取得件数を指定

```bash
docker compose exec app python main.py --video-id VIDEO_ID --comment-limit 50
```

### 4.3 複数動画の分析

```bash
docker compose exec app python main.py --video-ids VIDEO_ID1,VIDEO_ID2,VIDEO_ID3
```

### 4.4 再実行

再実行時は毎回YouTube APIからデータを取得して分析・表示する。データの永続化は行わない。

## 5. ログ設計

### 5.1 ログ出力先

| 種類 | 出力先 | 説明 |
|------|--------|------|
| アプリケーションログ | ./logs/app.log | バッチ処理の実行ログ（INFO以上） |
| エラーログ | ./logs/error.log | エラー・例外のみ（ERROR以上） |

### 5.2 ログフォーマット

```
[%(asctime)s] %(levelname)s %(name)s: %(message)s
```

出力例:
```
[2025-01-21 10:30:00] INFO fetch: Fetching video: abc123
[2025-01-21 10:30:01] INFO sentiment: Analyzed 10 comments
[2025-01-21 10:30:02] ERROR fetch: API quota exceeded
```

### 5.3 ログレベル

| レベル | 用途 |
|--------|------|
| DEBUG | 詳細なデバッグ情報 |
| INFO | 正常処理の記録 |
| WARNING | 警告（処理は継続） |
| ERROR | エラー（処理中断） |

### 5.4 ログローテーション

| 設定項目 | 値 |
|---------|-----|
| ローテーション単位 | 日次 |
| 保持期間 | 7日間 |
| 最大ファイルサイズ | 10MB |
| エンコーディング | UTF-8 |
| 圧縮 | gzip |

## 6. 監視項目

| 項目 | 閾値 | 対応 |
|------|------|------|
| APIクォータ使用量 | 8,000ユニット/日 | 警告ログ出力 |
| バッチ実行時間 | 30分超 | 警告ログ出力 |
| エラー発生 | 1件以上 | error.log確認 |

## 7. 障害対応

### 7.1 APIクォータ超過時

1. error.logで403エラーを確認
2. 翌日（太平洋時間0時）にクォータリセット
3. 処理動画数を減らして再実行

### 7.2 コンテナが起動しない

```powershell
# エラーログを確認
docker compose logs

# コンテナの状態を確認
docker compose ps

# コンテナとボリュームを再作成
docker compose down -v
docker compose up -d --build
```

### 7.3 ポートが使用中

```powershell
# 使用中のポートを確認
netstat -ano | findstr :8501

# docker-compose.ymlのポート設定を変更（例: 8501→8502）
```

### 7.4 ソースコードの変更が反映されない

```powershell
# Streamlitの場合: ブラウザで再読み込み

# Appコンテナの場合: コンテナを再起動
docker compose restart app

# それでも反映されない場合: 再ビルド
docker compose up --build -d
```

### 7.5 ディスク容量不足

```powershell
# 未使用のイメージとコンテナを削除
docker system prune -a

# ボリュームも含めて削除
docker system prune -a --volumes
```

```powershell

docker exec youtube-analytics-app ls -la /app

# コンテナからファイルをコピー
docker cp youtube-analytics-app:/app/logs/app.log ./logs/
```

## 9. セキュリティ

### 9.1 現在のセキュリティ設定

- コンテナ間通信は内部ネットワークのみ
- Streamlit（8501）は外部公開（開発環境用）
- APIキーは`.env`ファイルで管理（Git管理外）

### 9.2 本番環境での推奨事項

1. **Streamlitに認証を追加**する
2. **環境変数を安全に管理**する（`.env`はGit管理外、シークレット管理サービスの利用を検討）
3. **リソース制限を設定**する

```yaml
deploy:
  resources:
    limits:
      cpus: '2.0'
      memory: 2G
```

## 10. Azureクラウドデプロイ（Terraform）

### 10.1 構成概要

```
┌─────────────────────────────────────────────┐
│  Azure Resource Group                       │
│                                             │
│  ┌───────────────────┐                      │
│  │ Container Registry │ ← Docker イメージ   │
│  │ (ACR)             │   を Push            │
│  └────────┬──────────┘                      │
│           │ Pull                            │
│  ┌────────▼──────────────────────────────┐  │
│  │ Container Apps Environment            │  │
│  │  ┌─────────────────────────────────┐  │  │
│  │  │ Container App (Streamlit)       │  │  │
│  │  │  - Port 8501                    │  │  │
│  │  │  - YOUTUBE_API_KEY (Secret)     │  │  │
│  │  └─────────────────────────────────┘  │  │
│  │  ┌─────────────────────────────────┐  │  │
│  │  │ Log Analytics Workspace         │  │  │
│  │  └─────────────────────────────────┘  │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 10.2 使用するAzureサービス

| サービス | 用途 | 月額目安 |
|---------|------|---------|
| Azure Container Registry (Basic) | Dockerイメージ保管 | ~$5 |
| Azure Container Apps | Streamlitアプリ実行 | ~$0-10（従量課金） |
| Log Analytics Workspace | ログ収集 | 無料枠内 |

合計: 月額約 $5-15（低トラフィック時）

### 10.3 前提条件

```bash
# 必要なツール
az --version        # Azure CLI 2.50以上
terraform --version  # Terraform 1.5以上
docker --version     # Docker

# インストール（未導入の場合）
winget install Microsoft.AzureCLI
winget install Hashicorp.Terraform
```

### 10.4 Azureアカウント設定

```bash
# Azureにログイン
az login

# サブスクリプション確認
az account show

# サブスクリプションを指定（複数ある場合）
az account set --subscription "YOUR_SUBSCRIPTION_ID"
```

### 10.5 リソースプロバイダーの登録（初回のみ）

```bash
# 登録状態を確認
az provider show --namespace Microsoft.App --query "registrationState"
az provider show --namespace Microsoft.OperationalInsights --query "registrationState"

# "NotRegistered" の場合は登録
az provider register --namespace Microsoft.App
az provider register --namespace Microsoft.OperationalInsights
```

### 10.6 Terraformディレクトリ構成

```
terraform/
├── main.tf              # メインリソース定義
├── variables.tf         # 変数定義
├── outputs.tf           # 出力定義
├── terraform.tfvars     # 変数値（Git管理外）
└── .terraform.lock.hcl  # プロバイダーロック（自動生成）
```

### 10.7 terraform.tfvars の設定

```hcl
# このファイルはGit管理外にすること

# 必須設定
acr_name        = "youtubesentimentacr"   # グローバルで一意な名前に変更
youtube_api_key = "YOUR_YOUTUBE_API_KEY"

# オプション（デフォルト値を変更する場合のみ）
# subscription_id     = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
# resource_group_name = "rg-youtube-sentiment"
# acr_sku             = "Basic"
# location            = "japaneast"
# comment_limit       = 100
# container_cpu       = 2.0
# container_memory    = "4Gi"
```

### 10.8 デプロイ手順

#### Step 1: ACR（コンテナレジストリ）を先に作成

```bash
cd terraform
terraform init
terraform plan -target="azurerm_resource_group.main" -target="azurerm_container_registry.main"
terraform apply -target="azurerm_resource_group.main" -target="azurerm_container_registry.main"
```

#### Step 2: Dockerイメージのビルド・プッシュ

```bash
cd ..
az acr login --name youtubesentimentacr
docker build -t youtubesentimentacr.azurecr.io/youtube-sentiment:latest .
docker push youtubesentimentacr.azurecr.io/youtube-sentiment:latest
```

#### Step 3: 残りのリソースを作成

```bash
cd terraform
terraform plan
terraform apply
```

#### Step 4: 動作確認

```bash
terraform output app_url
# ブラウザでアクセス
```

### 10.9 更新デプロイ（コード変更時）

```bash
# 1. イメージを再ビルド
docker build -t youtubesentimentacr.azurecr.io/youtube-sentiment:latest .

# 2. ACRにプッシュ
az acr login --name youtubesentimentacr
docker push youtubesentimentacr.azurecr.io/youtube-sentiment:latest

# 3. Container App を更新
az containerapp update \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --image youtubesentimentacr.azurecr.io/youtube-sentiment:latest
```

### 10.10 環境変数の変更

```bash
# YouTube APIキーの変更
az containerapp secret set \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --secrets youtube-api-key=NEW_API_KEY

# リビジョンを再デプロイ
az containerapp update \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --image youtubesentimentacr.azurecr.io/youtube-sentiment:latest

# COMMENT_LIMITの変更
az containerapp update \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --set-env-vars COMMENT_LIMIT=200
```

### 10.11 リソースの削除

```bash
cd terraform
terraform destroy
```

### 10.12 Azureトラブルシューティング

#### コンテナが起動しない

```bash
az containerapp logs show \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --follow

az containerapp revision list \
  --name ca-youtube-sentiment \
  --resource-group rg-youtube-sentiment \
  --output table
```

#### メモリ不足エラー

`terraform.tfvars` の `container_memory` を増やす（推奨: 4Gi以上）。

#### ACRへのプッシュが失敗する

```bash
az acr login --name youtubesentimentacr
az acr credential show --name youtubesentimentacr
```

### 10.13 コスト管理

| 項目 | 条件 | 月額 |
|------|------|------|
| ACR Basic | 10GB ストレージ | ~$5 |
| Container Apps | min_replicas=0, 低トラフィック | ~$0-10 |
| Log Analytics | 無料枠 5GB/月 | $0 |
| **合計** | | **~$5-15** |

コスト削減のポイント:
- `min_replicas = 0` でアイドル時はコスト$0
- 使用しない時は `terraform destroy` で全削除
- ACR Basic SKU（$5/月）で十分
