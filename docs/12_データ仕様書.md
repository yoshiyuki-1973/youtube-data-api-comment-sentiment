# データ仕様書

YouTube動画コメント感情分析システム

## 1. データ方針

本システムではデータの永続化（ファイル保存・DB保存）は行わず、YouTube APIから取得したデータをメモリ上で処理し、結果を画面に表示する。実行のたびにYouTube APIからデータを取得し、分析・表示する。

## 2. データフロー

```
YouTube Data API v3
  │
  ├── videos.list → 動画メタデータ（1ユニット）
  │
  └── commentThreads.list → コメント一覧（1ユニット）
  │
  ▼
メモリ上のデータ構造（辞書・リスト）
  │
  ├── 動画情報 dict → fetch_video() の戻り値
  │
  ├── コメントリスト list[dict] → fetch_comments() の戻り値
  │
  ├── 感情分析済みコメントリスト list[dict] → classify_comments() の戻り値
  │
  └── 集計結果 dict → aggregate_video() の戻り値
  │
  ▼
画面表示（Streamlit UI / CLI出力）
```

## 3. データ構造

### 3.1 動画情報（fetch_video の戻り値）

```json
{
  "video_id": "string",
  "title": "string",
  "channel_id": "string",
  "channel_title": "string",
  "published_at": "string (ISO 8601)",
  "view_count": "integer",
  "like_count": "integer",
  "comment_count": "integer",
  "fetched_at": "string (ISO 8601)"
}
```

| フィールド | 型 | 説明 | APIフィールド |
|-----------|-----|------|-------------|
| video_id | str | 動画ID | id |
| title | str | 動画タイトル | snippet.title |
| channel_id | str | チャンネルID | snippet.channelId |
| channel_title | str | チャンネル名 | snippet.channelTitle |
| published_at | str | 公開日時（ISO 8601） | snippet.publishedAt |
| view_count | int | 再生回数 | statistics.viewCount |
| like_count | int | 高評価数 | statistics.likeCount |
| comment_count | int | コメント数 | statistics.commentCount |
| fetched_at | str | 取得日時（ISO 8601） | システム生成 |

### 3.2 コメント情報（fetch_comments の戻り値の各要素）

```json
{
  "comment_id": "string",
  "author": "string",
  "text": "string",
  "like_count": "integer",
  "published_at": "string (ISO 8601)"
}
```

| フィールド | 型 | 説明 | APIフィールド |
|-----------|-----|------|-------------|
| comment_id | str | コメントID | id |
| author | str | 投稿者名 | snippet.topLevelComment.snippet.authorDisplayName |
| text | str | コメント本文 | snippet.topLevelComment.snippet.textDisplay |
| like_count | int | いいね数 | snippet.topLevelComment.snippet.likeCount |
| published_at | str | 投稿日時（ISO 8601） | snippet.topLevelComment.snippet.publishedAt |

### 3.3 感情分析結果（classify_comment の戻り値）

```json
{
  "positive": "float (0.0-1.0)",
  "negative": "float (0.0-1.0)",
  "neutral": "float (0.0-1.0)",
  "language": "string (ja/other/unknown)"
}
```

| フィールド | 型 | 説明 | 範囲 |
|-----------|-----|------|------|
| positive | float | ポジティブスコア | 0.0 - 1.0 |
| negative | float | ネガティブスコア | 0.0 - 1.0 |
| neutral | float | ニュートラルスコア | 0.0 - 1.0 |
| language | str | 検出言語 | "ja" / "other" / "unknown" |

**制約**: `positive + negative + neutral = 1.0`（正規化済み）

### 3.4 感情分析済みコメント（classify_comments の戻り値の各要素）

```json
{
  "comment_id": "string",
  "author": "string",
  "text": "string",
  "like_count": "integer",
  "published_at": "string (ISO 8601)",
  "sentiment": {
    "positive": "float (0.0-1.0)",
    "negative": "float (0.0-1.0)",
    "neutral": "float (0.0-1.0)",
    "language": "string (ja/other/unknown)"
  }
}
```

元のコメント情報に `sentiment` フィールドが追加された形式。元のフィールドはすべて保持される。

### 3.5 集計結果（aggregate_video の戻り値）

```json
{
  "video_id": "string",
  "total_comments": "integer",
  "positive_count": "integer",
  "negative_count": "integer",
  "other_count": "integer",
  "positive_ratio": "float (4桁精度)",
  "negative_ratio": "float (4桁精度)",
  "positive_score": "float (4桁精度)",
  "negative_score": "float (4桁精度)",
  "neutral_score": "float (4桁精度)",
  "analyzed_at": "string (ISO 8601)"
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| video_id | str | 動画ID |
| total_comments | int | 分析対象コメント総数 |
| positive_count | int | ポジティブスコアが最大のコメント件数 |
| negative_count | int | ネガティブスコアが最大のコメント件数 |
| other_count | int | ニュートラル最大 or 同スコアのコメント件数 |
| positive_ratio | float | positive_count / total_comments（4桁精度） |
| negative_ratio | float | negative_count / total_comments（4桁精度） |
| positive_score | float | 全コメントの平均ポジティブスコア（4桁精度） |
| negative_score | float | 全コメントの平均ネガティブスコア（4桁精度） |
| neutral_score | float | 全コメントの平均ニュートラルスコア（4桁精度） |
| analyzed_at | str | 分析実行日時（ISO 8601） |

**計算ルール**:
- `positive_score + negative_score + neutral_score ≈ 1.0`（各コメントの平均）
- 同スコア（タイ）の場合は `other_count` にカウント
- コメント0件の場合は全フィールド0

### 3.6 分析結果（analyze_video の戻り値、Streamlit用）

```json
{
  "video": { "動画情報（3.1参照）" },
  "comments": [ "感情分析済みコメントリスト（3.4参照）" ],
  "summary": { "集計結果（3.5参照）" }
}
```

## 4. API仕様

### 4.1 YouTube Data API v3

#### videos.list

| パラメータ | 値 |
|-----------|-----|
| part | `snippet,statistics` |
| id | 動画ID |
| コスト | 1ユニット/リクエスト |

#### commentThreads.list

| パラメータ | 値 |
|-----------|-----|
| part | `snippet` |
| videoId | 動画ID |
| order | `relevance` |
| maxResults | 最大100（API_MAX_RESULTS環境変数） |
| pageToken | ページネーション用トークン |
| コスト | 1ユニット/リクエスト |

### 4.2 コメント取得のソート最適化

YouTube APIは直接いいね数順でのソートに対応していないため、以下の最適化を行う:

1. `relevance` 順で取得（いいね数との相関が高い）
2. `COMMENT_FETCH_MULTIPLIER`（デフォルト2）倍の件数を取得
3. メモリ上でいいね数降順にソート
4. 上位 `comment_limit` 件を返す

## 5. 環境変数によるデータ設定

| 変数名 | デフォルト | 説明 |
|--------|-----------|------|
| `COMMENT_LIMIT` | 10 | コメント取得件数 |
| `API_MAX_RESULTS` | 100 | 1リクエストあたり最大取得数 |
| `COMMENT_FETCH_MULTIPLIER` | 2 | ソート用取得倍率 |
