# ディレクトリ構成

YouTube動画コメント感情分析システム

## 1. 概要

本ドキュメントは、プロジェクトのディレクトリ構成を説明する。
本プロジェクトは **Docker実行に一本化** している。

## 2. プロジェクトルート構造

```
youtube-data-api-comment-sentiment/
├── .env                      # 環境変数設定ファイル（Git管理外）
├── .env.example              # 環境変数のテンプレート
├── .gitignore                # Git除外設定
├── docker-compose.yml        # Docker Compose設定
├── Dockerfile                # Dockerイメージ定義
├── pytest.ini                # pytest設定ファイル
├── requirements.txt          # Python依存パッケージ一覧
├── README.md                 # プロジェクト概要
├── CHANGELOG.md              # 変更履歴
├── CLAUDE.md                 # Claude Code設定
├── REVIEW_SUMMARY.md         # レビューサマリー
│
├── .vscode/                  # VSCode設定
├── .devcontainer/            # Dev Container設定
├── app/                      # アプリケーションコード
├── docs/                     # ドキュメント
├── logs/                     # ログファイル（Git管理外）
├── models/                   # 学習済みモデル（Git管理外）
├── terraform/                # Azureデプロイ用Terraform
└── tests/                    # テストコード
```

## 3. app/ - アプリケーションコード

```
app/
├── __init__.py               # パッケージ初期化
├── main.py                   # CLIバッチ処理エントリーポイント
├── streamlit_app.py          # Streamlit WebUIエントリーポイント
│
├── fetch/                    # YouTube API データ取得層
│   ├── __init__.py
│   └── youtube.py            # YouTube Data API クライアント
│
├── sentiment/                # 感情分析層
│   ├── __init__.py
│   └── analyzer.py           # 3-model BERTアンサンブル感情分析エンジン
│
└── aggregate/                # 集計層
    ├── __init__.py
    └── summarizer.py         # コメント集計・統計計算
```

### 各モジュールの役割

| モジュール | ファイル | 行数(概算) | 役割 |
|-----------|---------|-----------|------|
| **メイン** | `main.py` | 120行 | CLIバッチ処理（argparse） |
| | `streamlit_app.py` | 330行 | WebUI（Streamlit） |
| **fetch** | `youtube.py` | 220行 | YouTube Data API呼び出し、エラーハンドリング |
| **sentiment** | `analyzer.py` | 705行 | 3-model BERTアンサンブル + ルールベース(200+パターン) |
| **aggregate** | `summarizer.py` | 110行 | 感情スコア集計・統計 |

## 4. tests/ - テストコード

```
tests/
├── __init__.py               # パッケージ初期化
├── conftest.py               # pytest共通設定・フィクスチャ
├── test_fetch.py             # YouTube取得機能のテスト
├── test_analyzer.py          # 感情分析機能のテスト
├── test_summarizer.py        # 集計機能のテスト
├── test_integration.py       # 統合テスト
│
└── fixtures/                 # テスト用フィクスチャデータ
    ├── video_response.json   # 動画APIレスポンスのモック
    └── comments_response.json # コメントAPIレスポンスのモック
```

### テスト方針

- **ユニットテスト**: 各モジュールの単体機能テスト
- **統合テスト**: パイプライン全体のフロー確認
- **モックデータ**: `fixtures/` で外部API依存を排除
- **カバレッジ目標**: 全体80%以上、ビジネスロジック90%以上

## 10. Git管理外ファイル

`.gitignore` で以下を除外:

```gitignore
# 機密情報
.env
terraform/*.tfvars

# 自動生成・大容量
logs/
models/
__pycache__/
.pytest_cache/
*.pyc
terraform/.terraform/
terraform/*.tfstate
terraform/*.tfstate.backup
```

## 11. 設定ファイル

### .env（環境変数）

```bash
# 必須設定
YOUTUBE_API_KEY=your_api_key_here

# オプション設定
COMMENT_LIMIT=100
API_MAX_RESULTS=100
COMMENT_FETCH_MULTIPLIER=2
LOG_DIR=/app/logs
USE_FINETUNED_MODEL=false
FINETUNED_MODEL_PATH=./models/sentiment-finetuned
```
